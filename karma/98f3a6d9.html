HTML5发展

年份 大事记
1991 HTML
1994 HTML2
1996 CSS1+JavaScript
1997 HTML4
1998 CSS2
2000 XHTML
2002 Tableless Web Design（表格布局）
2005 AJAX
2009 HTML5
2014 HTLML5 Finalized

2002年的表格布局逐渐被淘汰；2009年就已经退出了HTML5的草案，直到2014年才有定稿，因为有移动端的推动。H5草案的前身叫：Web Application，最早由 WHATWG 这个组织在2004年提出。2007年被W3C组织接纳，并在2008-01-22发布HTML5的第一个草案。

HTML5作为HTML标记语言的最新版本，它制定了Web应用开发的一系列标准，成为第一个将Web作为应用开发平台的HTML语言。

HTML5广义概念：HTML5代表浏览器端技术的一个发展阶段。在这个阶段，浏览器的呈现技术得到了飞跃发展和广泛支持，它包括：HTML5、CSS3、JavaScript API在内的一套技术组合。是新一代开发Web富客户端（具有很强交互性和体验的客户端程序）应用程序的整体解决方案。

拖拽

在HTML5规范中，我们可以通过为元素增加draggable="true"来设置此元素是否可以进行拖拽操作，其中图片、链接默认开启拖拽功能。

  1. 拖拽元素

    页面中设置了draggable="true"属性的元素。

    拖拽元素的监听事件：

    • ondragstart当拖拽开始时调用
    • ondragleave当鼠标离开拖拽元素时调用
    • ondragend当拖拽结束时调用
    • ondrag整个拖拽过程都会调用

  2. 目标元素

    元素A拖拽到元素B，那么元素B就是目标元素。页面中任何一个元素都可以成为目标元素。

    目标元素的监听事件：

    • ondragenter当拖拽元素进入时调用
    • ondragover当拖拽元素停留在目标元素时，就会连续一直触发
    • ondrop当在目标元素上松开鼠标时调用
    • ondragleave当鼠标离开目标元素时调用

    如果想让拖拽元素在目标元素里做点事情，就必须要在 ondragover() 里加event.preventDefault()这一行代码。

地理位置

HTML5规范中，增加了获取用户地理信息的API，这样使得我们可以基于用户位置开发互联网应用，即基于位置服务LBS（Location Base Service）。

获取地理位置信息方式：

数据源 优点 缺点
IP地址 任何地方都可用，在服务器端处理 精确到城市级别（法规原因），运算代价大
Wi-Fi 精确
可在室内使用
简单、快捷		 Wi-Fi点介入少的地方无法使用
手机信号 相当精确
可在室内使用
简单、快捷		 需要设备能够访问
GPS 很精确 定位时间长、耗电量大
室内效果差
需要额外硬件设备支持
用户自定义 用户自行输入，比自动检测快 可能很不精确，当用户更换位置后

Html5 Geolocation规范提供了一套保护用户隐私的机制。必须先得到用户许可，才能获取用户的位置信息。

当成功获取地理信息后，会调用successCallback，并返回一个包含位置信息的对象position: Coords（即坐标）。当获取地理信息失败后，会调用errorCallback，并返回错误信息error。可选参数options对象可以调整位置信息数据的收集方式。

存储

随着互联网的快速发展，基于网页的应用越来越普遍，同时也变的越来越复杂，为了满足各种各样的需求，会经常性在本地存储大量的数据，传统方式我们以document.cookie来进行存储的，但是由于其存储大小只有4k左右，并且解析也相当的复杂，给开发带来诸多不便，HTML5规范则提出解决方案。

  1. 会话存储

    • 保存在内存中
    • 生命周期为关闭浏览器窗口
    • 在同一个窗口下数据共享

  2. 本地存储

    • 有可能在内存中，也有可能在硬盘中
    • 永久生效，可手动清理
    • 可以多窗口共享

相对于传统cookie方式，新的方式设置、读取更方便；容量较大（sessionStorage约5M、localStorage约20M）。两者在存储数据时，只能存储字符串，可以将对象JSON.stringfy()编码后存储。

应用缓存

HTML5中可以轻松的构建一个离线（无网络状态）应用，只需要创建一个 cache manifest 缓存清单文件。

作用：

  1. 可配置需要缓存的资源；
  2. 网络无连接应用仍可用；
  3. 本地读取缓存资源，提升访问速度，增强用户体验；
  4. 减少请求，缓解服务器负担。

缓存清单文件中列出了浏览器应缓存，以供离线访问的资源。推荐使用 .appcache作为后缀名，另外还要添加MIME类型。

缓存清单文格式：

  1. 顶行写CACHE MANIFEST
  2. CACHE: 换行 指定我们需要缓存的静态资源，如.css、image、js等
  3. NETWORK: 换行 指定需要在线访问的资源，可使用通配符
  4. FALLBACK: 换行 当被缓存的文件找不到时的备用资源

使用方式：

  1. 创建一个名为 demo.appcache的文件

  2. 在需要应用缓存在页面的根元素(html)里，添加属性manifest=”demo.appcache”。
